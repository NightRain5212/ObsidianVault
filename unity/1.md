
# 场景（Scene）

- 就是游戏中的一处场景，地点，关卡

# 游戏物体（Game Object）

- 即游戏中的道具，人物，建筑，道具等

# 3D视觉图

- 导航器（Gizmo），表示世界坐标的方向
- 栅格（Grid），表示xz平面
- 天空盒（Skybox），表示游戏世界的背景

- 旋转视图：alt+左键lmb
- 缩放试图：alt+右键rmb，滚轮
- 平移视图：中建mmb
- 恢复试图旋转：shift+左键导航器中间小方块
- 从某一个方向观察：左键导航器的轴

# 世界坐标系

- 3D游戏世界使用左手系。
- 一个物体的`transform`中的`position`就是这个物体所处的坐标。
- 一般y轴表示上下，xz轴表示东西南北。

# 视野中心

- 添加新物体时，默认在视野中心处创建。
- 旋转视角时，以视野中心为中心点旋转。
- 选中一个物体，按F键，可以将视野中心锁定该物体

# 透视，正交

- 透视试图：近大远小
- 正交试图：等距试图，物体的显示与距离无关
- 摄像机的广角field是可以设置的

# 平面

- unity的平面无厚度，且背面不会被渲染。正面可见，背面透明

# 材质修改

- 1. 材质重映射（Remap）
- 2. 将材质提取出来直接修改。
- 3. 创建模型时，可以不直接构建fbx模型，可以先创建对应的网格再添加对应的材质。

# 资源文件

- 模型文件Model（`*.fbx`)
- 图片文件Texture(`*.jpg/png/psd/tif`)
- 音频文件AudioClip(`*.mp3/wav/aiff`)
- 脚本文件Script（*`.cs`)
- 材质文件`*.mat`
- 场景文件`*.unity`: 包含了场景全局设置，以及各个节点，节点下的组件，组件参数
- 描述文件`*.meta`


# 轴心（Pivot）

- 轴心是一个物体的操作基准点。
- 物体的坐标一般表示其轴心点的位置。
- 移动旋转缩放时，以轴心为基准。
- 一般来说，轴心处于几何中心处。

# 父子节点

- 子节点会随父节点一起移动，删除时一并删除。
- 子节点是父节点的一部分。
- 子物体的坐标是相对父节点的

# 空物体

- 空物体没有网格，不可见。
- 空物体有坐标，可以移动。
## 组件

- **一个游戏对象（GameObject）本身只是一个空的容器，它的所有功能和行为都通过添加不同的组件（Component）来实现。**
- 游戏对象本身什么都不是，它的所有属性和行为都来源于它所挂载的组件。
- 一个简单的“玩家”对象可能就是由以下组件构成的：
	- `Transform` (默认就有)
	- `Mesh Renderer` (显示模型)
	- `Capsule Collider` (胶囊形碰撞体)
	- `Rigidbody` (物理刚体)
	- `PlayerMovement` (一个你自己写的，用于处理键盘输入和移动的脚本)
